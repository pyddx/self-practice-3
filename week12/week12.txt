
1. getProducts() — ดึงของทั้งหมดมาดู
ใช้ fetch(url) ยิง request ไปขอข้อมูล (ค่าเริ่มต้นคือ GET)
คำสั่ง await res.json() ทำหน้าที่แกะกล่อง แปลงข้อความ JSON ที่ได้มาให้เป็น Object หรือ Array ที่ JavaScript ใช้งานต่อได้
ใช้ console.log เพื่อเช็คของที่ได้
ครอบด้วย try/catch เสมอ เพื่อกันโปรแกรมพังเวลาเน็ตหลุดหรือข้อมูลผิดพลาด

2. addProduct() — สร้างรายการใหม่ (POST)
ต้องกำหนด method: "POST" เพื่อบอกว่าจะเพิ่มของ
สำคัญ: ต้องใส่ headers: { "Content-Type": "application/json" } เพื่อบอก Server ว่าของที่ส่งไปเป็น JSON นะ
ตัวข้อมูลใน body ห้ามส่งไปดื้อๆ ต้องห่อด้วย JSON.stringify(...) ให้เป็น String ก่อน
รอรับ response (res.json()) เพื่อดูผลลัพธ์ (ปกติจะได้ ID ของรายการใหม่กลับมา)

3. getProductById(id) — เจาะจงดูทีละอัน
ระบุเป้าหมายโดยการเอา id ไปต่อท้าย URL ตรงๆ
จุดต้องระวัง: fetch จะไม่ error ถ้าหาของไม่เจอ (404) เราเลยต้องเช็ค !res.ok เอง
ถ้าไม่เจอ ให้สั่ง throw new Error เพื่อบังคับให้ code กระโดดไปเข้า catch ทันที

4. updateProduct(id) — เปลี่ยนใหม่ยกชุด (PUT)
เป็นการ "วางทับ" ข้อมูลเดิม (Replace)
ต้องส่งข้อมูลไป ครบทุก field เหมือนตอนสร้างใหม่
เตือนความจำ: ถ้าส่งไปไม่ครบ field ที่หายไปอาจจะถูกลบทิ้งไปเลย (แล้วแต่ระบบหลังบ้านเขียนไว้)

5. patchProduct(id) — แก้ไขบางจุด (PATCH)
เป็นการ "ซ่อมแซม" เฉพาะส่วน (Partial Update)
ส่งไปแค่ค่าที่ต้องการแก้ (เช่น แก้แค่ราคา ก็ส่ง JSON ไปแค่ price)
Server จะทำการผสม (Merge) ค่าใหม่เข้าไป ของเดิมที่ไม่ได้แก้จะยังอยู่ครบ
เหมาะกับปุ่มเปิด/ปิด หรือแก้ไขฟอร์มแยกส่วน

6. deleteProduct(id) — ลบทิ้ง (DELETE)
ระบุ method: "DELETE" ไปที่ ID เป้าหมาย
เช็คความสำเร็จด้วย res.status ถ้าได้ 200 (OK) หรือ 204 (No Content) แปลว่าลบเสร็จสมบูรณ์
